### 오늘의 키워드

**1. 객체 지향**
- 객체들의 상호작용으로 파악
- 유연하고 변경이 용이함
- 추상화, 캡슐화, 상속, (다형성 -> 특정 역할에 대해 여러 구현 방식)
- 다형성 적용: 역할(인터페이스) 만들고 구현체(인터페이스를 구현한 클래스)를 바꾸기

**2. SOLID 원칙**
- 좋은 객체 지향 설계를 위한 5가지 원칙
- SRP 단일 책임 원칙: 클래스 1 -> 책임 1, 프로그램의 유지보수성을 높이기 위함
- OCP 개방 - 폐쇄 원칙: 확장에 열고 수정에 닫기, 기존 코드 수정 x 기능을 확장 가능
- LSP 리스코프 치환 원칙: 자식 클래스 -> 부모 클래스 대체 가능, 부모 클래스의 기능을 변경하면 안됨
- ISP 인터페이스 분리 원칙: 인터페이스를 사용에 따라 분리, 적합한 인터페이스 제공
- DIP 의존관계 역전 원칙: 구체화 x -> 추상화 o, 클래스 간의 결합도를 낮추기 위함
  
**3.의존관계 자동 주입**
- @Autowired -> 스프링 컨테이너에 등록된 빈 중 필요한 타입의 객체를 자동으로 주입
  1. 생성자 주입: 권장되는 방식, 객체의 주입을 강제, 불변성
  2. 필드 주입: 권장 x, 의존성 변경 가능성 있음
  3. 수정자 주입: 변경 가능성 있는 의존 관계에 사용, 선택적인 의존성 주입 ㄱㄴ
